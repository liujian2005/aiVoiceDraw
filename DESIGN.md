# VoiceDraw 设计文档

> 题目二：AI语音绘图工具 | 纯语音控制的绘图创作平台

---

## 一、系统架构设计

```
用户语音输入
     │
     ▼
┌─────────────────────┐
│   VoiceEngine        │  Web Speech API
│   语音识别引擎        │  连续/按钮双模式
└────────┬────────────┘
         │ 原始文字
         ▼
┌─────────────────────┐
│   CommandParser      │  两层架构
│   指令解析引擎        │  规则解析(快) + AI兜底(准)
└────────┬────────────┘
         │ 结构化指令
         ▼
┌─────────────────────┐
│   DrawEngine         │  HTML5 Canvas
│   绘图执行引擎        │  形状/场景/操作
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│   App (UI层)         │  状态反馈
│   主应用协调器        │  历史记录/Toast
└─────────────────────┘
```

## 三、最终实现情况

### ✅ 完全实现

- **语音识别**：基于 Web Speech API，支持中文识别，按住说话 + 空格键两种激活方式
- **指令解析引擎（双层）**：
  - 规则层：正则 + 关键词匹配，延迟 < 5ms，覆盖 95% 常见指令
  - AI层：调用 LLM API（DeepSeek/通义等 OpenAI 兼容接口），用于模糊指令兜底
- **9 种基础形状**：圆、矩形、三角、五角星、直线、菱形、心形、椭圆、箭头
- **9 种复合场景**：太阳、房子、树、笑脸、云、花、山、彩虹、汽车
- **颜色系统**：100+ 中文颜色名称，支持内嵌颜色词和显式设置
- **大小控制**：形容词档位 + 数字精确设置 + 相对放大/缩小
- **位置系统**：9宫格语义位置 + 光标移动指令
- **撤销/重做栈**：最多 20 步历史快照
- **实时反馈UI**：状态指示灯、中间结果显示、操作历史、Toast通知
- **虚拟光标**：可视化当前绘制位置

## 四、技术选型说明

### 为什么用 Web Speech API？
- **零依赖、免部署**：纯浏览器原生，不需要后端服务
- **中文支持好**：Chrome 的中文语音识别准确率优于大多数开源方案
- **低延迟**：识别结果几乎实时返回（中间结果 < 200ms）
- **缺点**：需要联网，Safari/Firefox 支持差，需 HTTPS（本地开发可绕过）

### 为什么规则解析优先于 AI？
- **响应速度**：规则解析 < 5ms vs AI API 调用 500ms-2s
- **离线可用**：规则层不依赖网络
- **可控性高**：精确控制每个指令的映射关系
- AI 作为兜底，仅在置信度低于阈值时触发，兼顾准确性与速度

### 为什么选 HTML5 Canvas？
- 轻量无依赖，原生支持所有绘图操作
- 历史快照（ImageData）轻松实现撤销/重做
- 方便导出为 PNG

## 五、容错与鲁棒性设计

1. **多候选择优**：语音识别返回多个候选结果，选择置信度最高的
2. **模糊匹配**：颜色/大小/位置按关键词长度降序匹配，避免短词截断长词
3. **AI兜底**：规则置信度 < 0.4 时自动调用 LLM 辅助理解
4. **错误恢复**：识别失败/API超时均有错误提示，不中断应用运行
5. **状态持久**：指令执行失败时不改变画布状态（通过 saveState 保护）

## 六、响应延迟分析

| 阶段 | 典型延迟 | 说明 |
|-----|---------|------|
| 语音识别（最终结果） | 300-800ms | 取决于语速和网络 |
| 规则指令解析 | < 5ms | 纯同步计算 |
| AI增强解析（可选） | 500-2000ms | 网络 + LLM推理 |
| Canvas绘图渲染 | < 10ms | 本地计算 |
| **总体响应** | **约 500ms-1s** | 规则路径（无AI时） |

## 七、扩展方向

1. **语音合成反馈（TTS）**：绘图完成后语音播报 "已绘制红色圆形"
2. **WebRTC协作绘图**：多人同时语音绘图
3. **本地 Whisper 识别**：替换 Web Speech API，支持离线识别
4. **指令学习**：记录用户习惯用法，优化规则权重
5. **导出多格式**：SVG、PDF、JSON指令序列回放
